## 블록 A — 학급 맞춤 뱃지·칭호 설정

### A-1. 목표

- 제작자가 등록한 전체 뱃지 풀(pool)에서 **교사가 학급에 사용할 뱃지를 선택**
- 뱃지 수에 따라 변하는 **칭호 이름·임계값을 교사가 직접 설정**
- 교사 대시보드에 **학급 설정** 탭 신설

### A-2. DB 스키마 변경

```sql
-- 학급별 활성 뱃지 설정
CREATE TABLE class_badge_settings (
  class_id   UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  badge_id   TEXT NOT NULL,
  is_enabled BOOLEAN NOT NULL DEFAULT true,
  PRIMARY KEY (class_id, badge_id)
);

-- 학급별 칭호 설정
CREATE TABLE class_title_settings (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id   UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  tier       INT  NOT NULL,          -- 1~5 (새싹→전설)
  name       TEXT NOT NULL,          -- 교사가 입력한 칭호 이름
  threshold  INT  NOT NULL,          -- 해당 칭호 진입 최소 뱃지 수
  UNIQUE(class_id, tier)
);
```

### A-3. 기본값 처리

- `class_badge_settings`에 행이 없으면 → 전체 뱃지 활성화 (기존 동작 유지)
- `class_title_settings`에 행이 없으면 → 기본 칭호(별빛 새싹 ~ 별빛 전설) 사용

### A-4. UI 구성 — 학급 설정 탭

```
교사 대시보드
└── [학급관리] [평가] [편지함] ... [학급설정 ← 신규]
    │
    ├── 뱃지 설정 섹션
    │   ├── 전체 뱃지 목록 (카테고리별 그리드)
    │   └── 각 뱃지 카드에 ON/OFF 토글
    │
    └── 칭호 설정 섹션
        ├── 5단계 칭호 이름 입력 필드
        ├── 각 칭호 진입 임계값(뱃지 수) 입력
        └── [저장] 버튼
```

### A-5. API 변경

| 메서드 | 경로                                 | 역할                     |
| ------ | ------------------------------------ | ------------------------ |
| GET    | `/api/teacher/class-settings`        | 학급 뱃지·칭호 설정 조회 |
| PUT    | `/api/teacher/class-settings/badges` | 활성 뱃지 목록 저장      |
| PUT    | `/api/teacher/class-settings/titles` | 칭호 설정 저장           |

### A-6. 뱃지·칭호 로직 변경 포인트

- `checkAndAwardBadge` → `class_badge_settings` 조회 후 비활성 뱃지는 skip
- `getTitleByBadgeCount` → class_id를 인자로 받아 `class_title_settings` 우선 적용
- `/api/badges/me` → 학생 소속 class의 설정을 반영해 응답

## 블록 B — 평가기준에 과목 추가 및 성취기준별 작성 통합

### B-1. 목표

- 평가기준(rubric) 생성 시 **과목명**을 함께 입력
- 학생 평가 대시보드에서 평가항목마다 **과목 태그** 표시
- 교사 대시보드 평가작성 탭에 **평가기준별 작성** 모드 신설
  - 기존의 "채점기준 고정 → 학생별 작성" 흐름을 성취기준별 작성으로 통합
  - 성취기준을 먼저 선택한 뒤, 전체 학생을 순서대로 평가하는 방식

### B-2. DB 스키마 변경

```sql
-- eval_rubrics 테이블에 과목 컬럼 추가
ALTER TABLE eval_rubrics ADD COLUMN IF NOT EXISTS subject TEXT;
-- 예: '국어', '수학', '사회', '과학', '영어', '음악', '미술', '체육', '도덕' 등

-- eval_report_items에 과목 스냅샷 추가 (평가 생성 시점 과목 보존)
ALTER TABLE eval_report_items ADD COLUMN IF NOT EXISTS rubric_subject_snapshot TEXT;
```

### B-3. UI 변경

#### 교사 — 평가기준 생성 폼

```
평가기준 만들기
├── 기준명                           (필수)
├── 과목  [국어 ▾]                   ← 신규 (datalist 자동완성)
├── 도달목표 / 수행과제
├── 평가기준 항목들
└── [채점기준 등록]
```

#### 교사 — 평가작성 탭 구조 변경

```
평가피드백
└── [평가 작성] [채점기준 관리]

    평가 작성 탭
    ├── [전체 현황] [학생별 작성] [성취기준별 작성 ← 신규]
    │
    ├── 전체 현황: 학생 카드 그리드 (기존 유지)
    │
    ├── 학생별 작성: 학생 선택 → 채점기준 선택 → 평가 입력 (기존 유지)
    │   (채점기준 고정 패널 제거 또는 성취기준별로 흡수)
    │
    └── 성취기준별 작성 ← 신규 (기존 채점기준 고정 방식 통합)
        ├── Step 1: 성취기준(채점기준) 선택
        │   ├── 과목 필터 탭: [전체] [국어] [수학] ...
        │   └── 채점기준 카드 그리드
        │
        └── Step 2: 선택한 성취기준으로 전체 학생 순차 평가
            ├── 상단: 선택된 성취기준 정보 (이름, 과목, 항목 요약)
            ├── 학생 목록 (좌측 고정, 완료 학생 체크 표시)
            └── 우측: 선택 학생의 평가 입력 폼
                ├── 채점기준 항목별 잘함/보통/노력 선택
                ├── 피드백 입력
                └── [저장 후 다음 학생 →] 버튼
```

#### 학생 — 평가 대시보드

```
내 평가 목록
├── [수학] 2학기 수학 단원평가   ← 과목 태그 표시
├── [국어] 독서감상문 루브릭
└── ...
```

### B-4. API 변경

- `POST /api/eval/rubrics` — `subject` 필드 추가
- `GET /api/eval/rubrics` — `subject` 포함 응답
- `POST /api/eval/reports` — `rubricSubjectSnapshot` 포함하여 저장
- `GET /api/eval/reports/my` — `rubric_subject_snapshot` 포함 응답

- [x] 채점기준 생성/수정 폼에 과목 필드 추가
- [x] 채점기준 목록에 과목 태그 표시
- [x] 채점기준 선택 패널에 과목 필터 탭 추가
- [x] 학생 평가 카드에 과목 태그 표시

## 블록 C — 생활기록부 탭

### C-1. 목표

- 교사 대시보드 평가 섹션에 **생활기록부** 탭 신설
- 채점기준을 선택하면 **전체 학생의 성취도와 피드백을 한 화면**에서 조회

### C-2. UI 구성 — 생활기록부 탭

```
교사 대시보드 → 평가 → [생활기록부 탭 ← 신규]
│
├── 상단: 채점기준(루브릭) 선택 드롭다운 (과목 필터 포함)
│
└── 채점기준별 학생 성적 테이블
    ┌──────────┬──────────────────┬───────────────────────────────┐
    │ 학생 이름 │ 항목별 성취도     │ 교사 피드백                    │
    ├──────────┼──────────────────┼───────────────────────────────┤
    │ 김철수    │ 잘함 / 보통 / 잘함│ "개념 이해 우수, 적용 연습 필요" │
    │ 이영희    │ 잘함 / 잘함 / 잘함│ "모든 영역 탁월한 수행"         │
    └──────────┴──────────────────┴───────────────────────────────┘
```

### C-3. API 엔드포인트

```
GET /api/teacher/school-record?rubricId={id}&classId={id}
Response: {
  rubric: { title, subject, goal, task, criteria[] },
  records: [
    { studentId, studentName, studentNumber, items: [{ criterionTitle, grade, feedback }] }
  ]
}
```

### C-4. DB 스키마 변경 없음

- 기존 `eval_reports`, `eval_report_items`, `eval_rubrics` 데이터를 조회해 구성
