## 블록 A — 학급 맞춤 뱃지·칭호 설정

### A-1. 목표

- 제작자가 등록한 전체 뱃지 풀(pool)에서 **교사가 학급에 사용할 뱃지를 선택**
- 뱃지 수에 따라 변하는 **칭호 이름·임계값을 교사가 직접 설정**
- 교사 대시보드에 **학급 설정** 탭 신설

### A-2. DB 스키마 변경

```sql
-- 학급별 활성 뱃지 설정
CREATE TABLE class_badge_settings (
  class_id   UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  badge_id   TEXT NOT NULL,
  is_enabled BOOLEAN NOT NULL DEFAULT true,
  PRIMARY KEY (class_id, badge_id)
);

-- 학급별 칭호 설정
CREATE TABLE class_title_settings (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id   UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  tier       INT  NOT NULL,          -- 1~5 (새싹→전설)
  name       TEXT NOT NULL,          -- 교사가 입력한 칭호 이름
  threshold  INT  NOT NULL,          -- 해당 칭호 진입 최소 뱃지 수
  UNIQUE(class_id, tier)
);
```

### A-3. 기본값 처리

- `class_badge_settings`에 행이 없으면 → 전체 뱃지 활성화 (기존 동작 유지)
- `class_title_settings`에 행이 없으면 → 기본 칭호(별빛 새싹 ~ 별빛 전설) 사용

### A-4. UI 구성 — 학급 설정 탭

```
교사 대시보드
└── [학급관리] [평가] [편지함] ... [학급설정 ← 신규]
    │
    ├── 뱃지 설정 섹션
    │   ├── 전체 뱃지 목록 (카테고리별 그리드)
    │   └── 각 뱃지 카드에 ON/OFF 토글
    │
    └── 칭호 설정 섹션
        ├── 5단계 칭호 이름 입력 필드
        ├── 각 칭호 진입 임계값(뱃지 수) 입력
        └── [저장] 버튼
```

### A-5. API 변경

| 메서드 | 경로 | 역할 |
|--------|------|------|
| GET | `/api/teacher/class-settings` | 학급 뱃지·칭호 설정 조회 |
| PUT | `/api/teacher/class-settings/badges` | 활성 뱃지 목록 저장 |
| PUT | `/api/teacher/class-settings/titles` | 칭호 설정 저장 |

### A-6. 뱃지·칭호 로직 변경 포인트

- `checkAndAwardBadge` → `class_badge_settings` 조회 후 비활성 뱃지는 skip
- `getTitleByBadgeCount` → class_id를 인자로 받아 `class_title_settings` 우선 적용
- `/api/badges/me` → 학생 소속 class의 설정을 반영해 응답

## 블록 B — 평가기준에 과목 추가 및 과목별 정렬

### B-1. 목표

- 평가기준(rubric) 생성 시 **과목명**을 함께 입력
- 학생 평가 대시보드에서 평가항목마다 **과목 태그** 표시
- 교사 대시보드 평가 탭에서 **과목별 필터 및 정렬** 제공

### B-2. DB 스키마 변경

```sql
-- eval_rubrics 테이블에 과목 컬럼 추가
ALTER TABLE eval_rubrics ADD COLUMN IF NOT EXISTS subject TEXT;
-- 예: '국어', '수학', '사회', '과학', '영어', '음악', '미술', '체육', '도덕' 등
```

### B-3. UI 변경

#### 교사 — 평가기준 생성 폼

```
평가기준 만들기
├── 과목 선택 (드롭다운 또는 텍스트 입력)   ← 신규
├── 평가명
├── 채점기준 항목들
└── [저장]
```

#### 학생 — 평가 대시보드

```
내 평가 목록
├── [수학] 2학기 수학 단원평가   ← 과목 태그 표시
├── [국어] 독서감상문 루브릭
└── ...
```

#### 교사 — 평가 탭

```
평가 관리
├── 과목 필터 탭: [전체] [국어] [수학] [사회] ...  ← 신규
├── 정렬: 최신순 / 과목순 / 학생순
└── 평가 목록 (과목 태그 포함)
```

### B-4. API 변경

- `POST /api/eval/rubrics` — `subject` 필드 추가
- `GET /api/eval/rubrics` — `?subject=수학` 쿼리 파라미터로 필터
- `GET /api/teacher/students/[id]/evals` — 응답에 `subject` 포함

## 블록 C — 생활기록부 탭 + AI 성취도 자동생성

### C-1. 목표

- 교사 대시보드 평가 섹션에 **생활기록부** 탭 신설
- 채점기준별로 **전체 학생 성적을 한 화면**에서 조회
- Claude AI를 활용해 채점기준 + 교사 피드백 + 학생 성취도를 종합한 **생기부 문장 자동생성**

### C-2. UI 구성 — 생활기록부 탭

```
교사 대시보드 → 평가 → [생활기록부 탭 ← 신규]
│
├── 상단: 채점기준(루브릭) 선택 드롭다운
│
├── 채점기준별 학생 성적 테이블
│   ┌──────────┬───────┬───────────────────────────────┐
│   │ 학생 이름 │ 점수  │ 교사 피드백                    │
│   ├──────────┼───────┼───────────────────────────────┤
│   │ 김철수    │  4/5  │ "개념 이해 우수, 적용 연습 필요" │
│   │ 이영희    │  5/5  │ "모든 영역 탁월한 수행"         │
│   └──────────┴───────┴───────────────────────────────┘
│
├── 하단 섹션: AI 생기부 생성
│   ├── 학생 선택 (드롭다운 또는 카드 클릭)
│   ├── [AI 생기부 문장 생성] 버튼
│   ├── 생성 결과 텍스트 박스 (편집 가능)
│   └── [복사] [저장] 버튼
```

### C-3. AI 생기부 생성 로직

#### 입력 데이터 (프롬프트 컨텍스트)

```
- 채점기준명 및 각 항목 설명
- 학생의 항목별 점수
- 교사가 입력한 피드백 텍스트
- 학생의 성찰일기 내용 (선택적 포함)
```

#### 프롬프트 구조 (pseudo)

```
당신은 초등학교/중학교 교사입니다.
아래 정보를 바탕으로 학교생활기록부에 들어갈 학생 성취 문장을 작성해주세요.
- 200자 내외, 긍정적이고 구체적인 서술
- 수행 수준과 개선 가능성을 함께 언급

[채점기준]
{rubric 항목별 설명}

[학생 성취 점수]
{항목별 점수}

[교사 피드백]
{feedback_text}
```

#### API 엔드포인트

```
POST /api/teacher/school-record/generate
Body: { studentId, rubricId }
Response: { generatedText: string }
```

- Claude API (`claude-sonnet-4-6`) 사용
- 스트리밍 응답으로 타이핑 효과 구현 (선택)
- 생성 결과를 `school_records` 테이블에 저장 가능

### C-4. DB 스키마 (선택적 저장)

```sql
CREATE TABLE school_records (
  id           UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id   UUID        NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  rubric_id    UUID        NOT NULL REFERENCES eval_rubrics(id) ON DELETE CASCADE,
  generated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  content      TEXT        NOT NULL,  -- AI 생성 또는 교사 편집 최종본
  edited       BOOLEAN     NOT NULL DEFAULT false
);
```
