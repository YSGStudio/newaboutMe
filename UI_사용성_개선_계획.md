# 마음일기 UI 사용성 개선 계획서

작성일: 2026-02-27  
대상: `app/teacher/page.tsx`, `app/student/page.tsx`, 공통 스타일(`app/globals.css`)

---

## 1. 목적
현재 MVP UI는 기능 검증에는 충분하지만, 실제 교사/학생 사용 시 아래 불편이 있다.

- 한 화면에 입력 요소가 많아 핵심 행동(작성/등록/체크) 집중이 어렵다.
- 상태 피드백(성공/실패/로딩)이 약해 사용자 확신이 낮다.
- 모바일에서 터치 영역/정보 밀도가 최적화되어 있지 않다.

이번 개선의 목표는 **빠르게 이해되고, 실수 없이, 반복 사용하기 쉬운 UI**로 만드는 것이다.

---

## 2. 핵심 개선 원칙

1. 첫 화면 5초 이해
- 각 화면의 1순위 액션을 화면 상단에 배치한다.

2. 입력 부담 최소화
- 한 번에 하나의 작업만 하게 단계화한다.

3. 피드백 즉시 제공
- 로딩, 성공, 실패, 빈 상태를 명확히 노출한다.

4. 모바일 우선
- 학생 화면은 모바일 사용을 기준으로 설계한다.

5. 접근성 확보
- 대비, 폰트 크기, 키보드 포커스, aria 라벨 기본 적용.

---

## 3. 우선순위 백로그

## P0 (즉시)
- 교사/학생 페이지 상단에 고정 헤더 + 현재 사용자 정보 표시
- 모든 폼 제출 버튼에 로딩 상태(`처리 중...`) 추가
- 에러 메시지 영역을 필드 근처에 표시
- 성공 메시지 `setTimeout` 자동 사라짐 처리
- 빈 상태 UI(데이터 없음 안내 + 다음 행동 버튼) 개선

## P1 (단기)
- 학생 홈을 3탭 구조로 분리: `오늘의 감정 / 오늘의 계획 / 감정 타임라인`
- 교사 화면을 2열 대시보드에서 `학급 관리 / 학생 관리` 탭 구조로 전환
- 입력 컴포넌트 공통화: `FormField`, `SubmitButton`, `Notice`
- 감정/반응 버튼 터치 영역 44px 이상 확보

## P2 (중기)
- 계획 완료 시 마이크로 인터랙션(체크 애니메이션)
- 타임라인 카드 정보 계층 정리(작성자, 시간, 본문, 반응)
- 월간 통계 화면 프리뷰(placeholder 차트 UI)

## 4. 화면별 개선안

## 4.1 교사 화면 (`/teacher`)

현재 이슈:
- 인증, 학급 생성, 학생 등록이 동시에 보여 초점 분산

개선:
- 상단: `학급 선택 드롭다운 + 학급코드 + 로그아웃`
- 탭:
  - 탭1 `학급 관리`: 학급 생성 폼 + 학급 목록 카드
  - 탭2 `학생 관리`: 학생 등록 폼 + 학생 테이블
- 학생 목록은 헤더 고정 테이블로 변경
- PIN은 기본 마스킹 + “보기” 토글 제공

## 4.2 학생 화면 (`/student`)

현재 이슈:
- 로그인 이후 정보량이 많아 집중 분산

개선:
- 로그인 전/후 레이아웃 분리
- 로그인 후 상단에 오늘 요약 카드 고정
  - 오늘 감정 작성 수 (0/3)
  - 계획 달성률
- 하단 탭 네비게이션
  - `감정 작성`
  - `계획 체크`
  - `타임라인`
- 계획 체크는 카드형 + 큰 체크박스 + 진행률 바

## 5. 컴포넌트 구조 개선

추가 권장 컴포넌트:
- `components/ui/PageHeader.tsx`
- `components/ui/Notice.tsx`
- `components/ui/SubmitButton.tsx`
- `components/ui/EmptyState.tsx`
- `components/ui/Tabs.tsx`
- `components/ui/ProgressBar.tsx`

스타일 정책:
- 색상/간격/라운드 값을 CSS 변수로 통일
- 버튼 스타일 3종(`primary`, `secondary`, `ghost`) 표준화
- 폼 에러 텍스트/힌트 텍스트 토큰 통일

---

## 6. 반응형/접근성 체크리스트

반응형:
- `<= 768px`: 단일 컬럼
- `> 768px`: 교사 페이지 2컬럼 허용
- 하단 고정 CTA가 키보드에 가리지 않도록 처리

접근성:
- 라벨 없는 입력 금지
- 포커스 링 항상 표시
- 텍스트 최소 14px, 주요 본문 16px
- 색상 대비 WCAG AA 준수

## 8. 완료 기준 (Definition of Done)

- 신규 사용자가 1분 내 핵심 행동 수행 가능
  - 교사: 학급 생성 + 학생 등록
  - 학생: 로그인 + 감정 작성 + 계획 체크
- 폼 제출 중 중복 클릭 방지
- 오류 메시지 원인 명확히 표시
- 모바일(아이폰/안드로이드)에서 레이아웃 깨짐 없음
- `npm run lint`, `npm run build` 통과
