# 마음일기 AI 성장 리포트 고도화 계획서

> 작성일: 2026.06.29 | 버전: v3.0 (gpt-4o 모델 / UI 진입점 확정)

---

## 1. 개요

### 1.1 목적

기존 성장 리포트(AboutMe 보고서)는 계획 달성률·감정 분포·평가 현황을 수치와 그래프로만 보여주었다. 이를 OpenAI gpt-4o 모델과 결합하여 교사가 직접 작성하기 어렵던 **서술형 교과발달상황**을 자동으로 생성하고, 감정 패턴 분석과 맞춤 성장 제언까지 포함한 고품질 리포트를 제공한다. 분석 결과는 모두 **PDF로 저장**할 수 있다.

### 1.2 사용 모델

| 항목 | 내용 |
|------|------|
| **모델** | `gpt-4o` (고정) |
| **API 제공사** | OpenAI |
| **환경변수** | `OPENAI_API_KEY` |

> **모델 선택 근거:** 교과발달상황은 학교생활기록부에 준하는 정밀한 서술이 필요하므로 gpt-4o를 고정 사용한다. gpt-4o-mini로의 다운그레이드는 허용하지 않는다.

### 1.3 비용 (gpt-4o 기준)

| 항목 | 내용 |
|------|------|
| 입력 단가 | $2.50 / 1M 토큰 |
| 출력 단가 | $10.00 / 1M 토큰 |
| 회당 예상 토큰 | 입력 ~2,400 / 출력 ~700 |
| **회당 비용** | **~$0.013 (약 18원)** |
| **1,000회 비용** | **~$13 (약 18,000원)** |
| 캐싱 적용 후 실질 비용 | 학급 30명 기준 한 학기 약 **수백 원 수준** |

## 2. ⚠️ 개인정보 보호 원칙 (필수 준수)

> **이 섹션은 개발 착수 전 반드시 검토하고 코드 리뷰 체크리스트에 포함해야 한다.**

### 2.1 학생 이름 — OpenAI에 절대 전송 금지

실제 학생 이름은 API 요청 어디에도 포함해서는 안 된다. 프롬프트 내에서 학생을 지칭할 때는 **출석번호 기반 코드**로만 표기한다.

```typescript
// ❌ 절대 금지
const prompt = `김민준 학생의 평가 결과입니다...`;

// ✅ 올바른 방식
const anonymizedLabel = `${student.studentNumber}번 학생`;
const prompt = `${anonymizedLabel}의 평가 결과입니다...`;

// 응답을 화면/PDF에 표시할 때만 실제 이름으로 치환
const displayResult = aiResponse.replace(
  `${student.studentNumber}번 학생`,
  student.name
);
```

치환 흐름:
```
실제 이름 (DB)
    │
    ▼  [서버에서 익명화]
"7번 학생" (GPT 전송)
    │
    ▼  [GPT 응답 수신 후 서버에서 복원]
실제 이름 (화면/PDF 표시)
```

### 2.2 OpenAI 모델 학습 — 반드시 opt-out

```typescript
// ✅ 모든 OpenAI 호출에 반드시 포함
const response = await openai.chat.completions.create({
  model: 'gpt-4o',
  store: false,   // ← 이 옵션 없으면 배포 불가 / 코드 리뷰 반려
  messages: [
    { role: 'system', content: SYSTEM_PROMPT },
    { role: 'user', content: userPrompt }
  ]
});
```

### 2.3 OpenAI로 전송하는 데이터 범위

| 항목 | 전송 여부 | 이유 |
|------|-----------|------|
| 학생 실명 | ❌ 금지 | 출석번호 코드로 대체 |
| 학생 DB ID / 학급 코드 | ❌ 금지 | 불필요한 식별자 |
| 감정 원문 텍스트 | ✅ 전송 | 패턴 분석에 필수 |
| 감정 범주 / 세부 감정 | ✅ 전송 | 필수 |
| 날짜 → 상대 표현 | ✅ 전송 | "3주차 월요일" 형식 변환 |
| 계획 제목 / 달성률 | ✅ 전송 | 필수 |
| 루브릭 제목 / 목표 / 기준 / 등급 / 피드백 | ✅ 전송 | 교과발달상황 생성 필수 |
| 학생 성찰일기 | ⚠️ 선택 | 민감 가능, 요약본만 전송 권장 |
| 부모님 응원 내용 | ❌ 금지 | 학부모 개인정보 |
| 이미지 파일 | ❌ 금지 | 평가 자료 이미지 전송 불가 |

## 3. AI가 생성하는 3가지 결과물

### 3.1 교과발달상황

루브릭 제목 기준 과목별 서술형 발달 상황 (100~150자).

> "국어 말하기·듣기 영역에서 내용 전달력과 발표 자세가 우수하며, 청중을 배려하는 표현 방식을 잘 활용하고 있습니다. 목소리 크기는 보통 수준으로, 좀 더 자신감 있게 발표할 수 있도록 꾸준한 연습이 필요합니다."

### 3.2 감정 패턴 인사이트

감정 원문·범주·날짜(상대 표현)를 분석한 시계열 패턴 요약.

> "3주차에 불안/긴장 관련 기록이 집중되어 있으며 '시험', '발표' 키워드가 반복됩니다. 이후 기쁨/활력 비율이 회복되어 전반적으로 긍정적인 흐름을 유지하고 있습니다."

### 3.3 맞춤 성장 제언

계획 달성률·감정 패턴·평가 결과를 종합한 구체적 다음 단계 제안.

> "'수학 문제집 풀기' 항목의 달성률이 40%로 낮습니다. 분량을 줄이거나 요일을 지정하는 방식으로 계획을 조정하면 지속 가능성이 높아질 것입니다."

## 4. UI 진입점 및 기능 흐름

### 4.1 진입점 A — 학급 전체 분석 (성장 리포트 탭)

**위치:** 교사 대시보드 > 성장 리포트 탭 > 상단 버튼 영역

```
[ 전체 내보내기 ]  [ 전체 분석하기 ★ ]      ← 버튼 나란히 배치
```

**동작 흐름:**

```
교사가 [전체 분석하기] 클릭
    │
    ▼
기간 필터 확인 (주간/월간/학기 현재 선택값 사용)
    │
    ▼
전체 학생 수 안내 + 확인 다이얼로그
  "30명 학생의 AI 분석을 생성합니다. 약 N분 소요됩니다."
    │ 확인
    ▼
학생별 AI 분석 API 병렬 호출 (Promise.allSettled)
  - 캐시 있는 학생: 즉시 반환
  - 캐시 없는 학생: gpt-4o 호출
    │
    ▼
진행률 표시 (예: "분석 중... 12/30")
    │
    ▼
전체 완료 → PDF 자동 생성
  - 표지 (학급명, 날짜, 학생 수)
  - 학생별 페이지: 기존 AboutMe 내용 + AI 분석 섹션
    │
    ▼
PDF 다운로드 (파일명: 학급명_AI성장리포트_날짜.pdf)
```

**에러 처리:** 일부 학생 분석 실패 시 해당 학생 페이지에 "AI 분석을 불러올 수 없습니다" 표시 후 나머지 학생은 정상 출력.

### 4.2 진입점 B — 학생 개별 분석 (AboutMe 보고서 모달)

**위치:** 성장 리포트 탭 > 학생 카드 클릭 > AboutMe 보고서 모달 > 상단 버튼 영역

```
[ PDF 저장 ]  [ 분석하기 ★ ]      ← 버튼 나란히 배치
```

**동작 흐름:**

```
교사가 [분석하기] 클릭
    │
    ▼

  - 캐시 있음: 즉시 AI 섹션 표시
  - 캐시 없음: 로딩 Skeleton → gpt-4o 호출 (약 3~5초)
    │
    ▼
모달 하단 ④ AI 성장 분석 섹션 렌더링
```

## 6. 신규 API 설계

### POST `/api/ai/growth-report/[studentId]`

**요청:**
```json
{ "period": "month" }
```

**응답:**
```json
{
  "subjectReports": [
    { "subject": "발표 태도", "content": "국어 말하기·듣기 영역에서..." }
  ],
  "emotionInsight": "3주차에 불안/긴장 관련 기록이 집중되어...",
  "growthSuggestion": "'수학 문제집 풀기' 항목의 달성률이 40%로...",
  "generatedAt": "2026-06-29T10:00:00Z",
  "cached": false
}
```

**처리 순서:**
1. 교사 인증 확인 (`requireTeacher`)
2. 학생이 해당 교사 학급 소속인지 검증
3. 당일 캐시 확인 → 있으면 즉시 반환
4. 데이터 수집 (snapshot + evalReports)
5. 익명화 처리 (이름 제거, 날짜 상대 표현 변환)
6. gpt-4o 호출 (`store: false` 필수)
7. 응답 파싱 → DB 저장 → 반환

### POST `/api/ai/growth-report/class` (전체 분석용)

**요청:**
```json
{ "classId": "...", "period": "month" }
```

**응답:**
```json
{
  "results": [
    { "studentId": "...", "status": "success", "cached": true },
    { "studentId": "...", "status": "success", "cached": false },
    { "studentId": "...", "status": "error", "message": "..." }
  ],
  "total": 30,
  "succeeded": 29,
  "failed": 1
}
```

> 내부적으로 학생별 분석을 병렬 호출하되 OpenAI rate limit을 고려해 최대 5개씩 병렬 처리.

## 7. GPT 프롬프트 설계

### 시스템 프롬프트

```
당신은 대한민국 초등학교 담임교사의 업무를 보조하는 AI입니다.
교사가 수집한 학생 평가 데이터를 바탕으로 학교생활기록부 수준의
정확하고 따뜻한 교과발달상황을 작성합니다.

작성 원칙:
- 초등학생 눈높이에 맞는 따뜻하고 격려하는 어조 사용
- 학교생활기록부 서술형 표현 양식 준수 (3인칭 서술, 과거형)
- 교과별 발달상황은 100~150자 내외로 간결하게 작성
- 부정적 표현 대신 성장 가능성 중심으로 서술
- 반드시 JSON 형식으로만 응답할 것

중요: 입력 데이터에 학생의 실명이 포함되지 않습니다.
출력에서도 "N번 학생" 표현을 그대로 유지하세요.
```

### 유저 프롬프트 구조

```
다음은 7번 학생의 최근 1개월 성장 데이터입니다.

=== 계획 달성 현황 ===
전체 달성률: 72%
- 책 30분 읽기: 90% 달성
- 수학 문제집 5쪽: 40% 달성

=== 감정 기록 (최근 1개월) ===
총 22건
- 1주차 월요일 / 불안/긴장 / 걱정 / "시험이 너무 걱정돼요"
- 1주차 수요일 / 기쁨/활력 / 뿌듯함 / "발표 잘됐어요"
...

=== 평가 보고서 ===
[발표 태도 평가]
  도달목표: 청중을 배려하며 내용을 전달한다
  수행과제: 3분 스피치 발표
  - 내용 전달력: 잘함 — "논리적으로 구성하여 발표함"
  - 목소리 크기: 보통 — "좀 더 크게 말하면 좋겠음"

위 데이터를 바탕으로 다음 JSON을 생성해주세요:
{
  "subjectReports": [
    { "subject": "루브릭 제목", "content": "100~150자 교과발달상황" }
  ],
  "emotionInsight": "감정 패턴 분석 요약 (2~3문장)",
  "growthSuggestion": "맞춤 성장 제언 (2~3문장)"
}
```

## 9. 캐싱 전략

| 기준 | 내용 |
|------|------|
| 캐시 키 | `studentId + period + 날짜(YYYY-MM-DD)` |
| 저장 위치 | Supabase `ai_growth_reports` 테이블 (신규) |
| 유효 기간 | 당일 자정까지 |
| 재분석 조건 | 기간 필터 변경 또는 교사가 직접 "재분석" 클릭 |

### ai_growth_reports 테이블 (신규)

```sql
CREATE TABLE ai_growth_reports (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID REFERENCES students(id) ON DELETE CASCADE,
  teacher_id UUID REFERENCES teachers(id),
  period TEXT NOT NULL,              -- 'week' | 'month' | 'semester'
  generated_date DATE NOT NULL,      -- 생성 날짜 (캐시 기준)
  subject_reports JSONB NOT NULL,    -- 교과발달상황 배열
  emotion_insight TEXT NOT NULL,     -- 감정 패턴 인사이트
  growth_suggestion TEXT NOT NULL,   -- 맞춤 성장 제언
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE (student_id, period, generated_date)
);
```

## 12. 홀란드 기반 성향 분석 (Holland Career Aptitude)

> 추가 예정 기능 | 버전: v4.0

### 12.1 개요

학생의 계획 실천 패턴·감정 기록·과목별 평가 데이터를 바탕으로 **홀란드 직업 성격 이론(RIASEC)** 의 6가지 유형 중 두드러지는 성향을 gpt-4o가 추론하고, 관련 추천 직업군을 함께 제공한다. 정식 심리 검사가 아닌 **학생 기록 기반 성향 힌트**로 위치시킨다.

### 12.2 홀란드 6유형 참조

| 유형 | 명칭 | 특징 키워드 |
|------|------|-------------|
| R | 현실형 (Realistic) | 신체 활동, 도구, 기계, 야외 |
| I | 탐구형 (Investigative) | 분석, 호기심, 과학, 독립적 사고 |
| A | 예술형 (Artistic) | 창의성, 표현, 감수성, 자유 |
| S | 사회형 (Social) | 협력, 도움, 대화, 공감 |
| E | 진취형 (Enterprising) | 리더십, 설득, 목표 지향 |
| C | 관습형 (Conventional) | 체계, 규칙, 정확성, 계획 실천 |

### 12.3 데이터 → 유형 연결 근거

| 현재 데이터 | 유형 추론 근거 |
|---|---|
| 체계적 계획 달성률 높음 | C형 성향 |
| 독서·과학 계획 달성률 높음 | I형 성향 |
| 감정: 사회적 감정 비중 높음 | S형 성향 |
| 감정: 표현·예술 관련 기록 | A형 성향 |
| 모둠 리더 역할에서 성취감 | E형 성향 |
| 체육·신체 활동 계획 달성률 높음 | R형 성향 |
| 국어·사회 성취도 높음 | S·A형 보강 |
| 수학·과학 성취도 높음 | I·R형 보강 |

### 12.5 GPT 프롬프트 설계

**시스템 프롬프트 핵심:**

```
당신은 대한민국 초등학교 담임교사를 보조하는 AI입니다.
학생의 행동 데이터(계획 실천, 감정 기록, 과목별 성취)를 바탕으로
홀란드 직업 성격 이론(RIASEC)의 6유형 중 이 학생에게 두드러지는
성향 1~2가지를 추론합니다.

주의사항:
- 정식 심리검사가 아닌 기록 기반 참고 추론임을 전제
- 부정적 표현 없이 강점 중심으로 서술
- 초등학생 눈높이에 맞는 따뜻한 어조
- 반드시 JSON 형식으로만 응답
```

**출력 스키마:**

```json

  "primaryType": "I",
  "primaryTypeLabel": "탐구형",
  "primaryReason": "독서와 과학 분야에서 높은 달성률과 성취를 보이며...",
  "secondaryType": "C",
  "secondaryTypeLabel": "관습형",
  "secondaryReason": "루틴이 명확한 계획일수록 달성률이 높고...",

  "caveat": "이 결과는 학생 기록 데이터를 바탕으로 한 AI 참고 추론으로, 정식 직업 적성 검사를 대체하지 않습니다."

```

### 12.6 데이터 최소 요건

| 조건 | 기준 |
|------|------|
| 감정 기록 | 10건 이상 |
| 계획 실천 기록 | 1개 이상의 계획, 7일 이상 기간 |
| 평가 기록 | 없어도 분석 가능 (있으면 정확도 향상) |

→ 미충족 시 "AI 생성" 버튼 비활성화 + "기록이 쌓이면 분석할 수 있어요" 안내

### 12.7 저장 전략

- 성향 분석 결과는 **별도 테이블 `ai_holland_reports`** 에 `student_id` 기준 1건만 영구 저장 (종합평가와 동일 방식)
- "AI 생성" 버튼 재클릭 시 덮어쓰기
- PDF 저장 시 About Me 모달 내 ⑤섹션도 포함

```sql
CREATE TABLE IF NOT EXISTS ai_holland_reports (
  id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id      UUID NOT NULL UNIQUE REFERENCES students(id) ON DELETE CASCADE,
  teacher_id      UUID NOT NULL REFERENCES teacher_profiles(id) ON DELETE CASCADE,
  primary_type    TEXT NOT NULL,         -- 'R'|'I'|'A'|'S'|'E'|'C'
  primary_label   TEXT NOT NULL,
  primary_reason  TEXT NOT NULL,
  secondary_type  TEXT,
  secondary_label TEXT,
  secondary_reason TEXT,
  career_suggestions JSONB NOT NULL,     -- string[]
  generated_at    TIMESTAMPTZ NOT NULL DEFAULT now(),
  created_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);
ALTER TABLE ai_holland_reports ENABLE ROW LEVEL SECURITY;
```

### 12.10 한계 및 주의사항

- 초등학생 대상 홀란드 적용은 문헌 근거 부족 — 어디까지나 **보조 참고 자료**
- 데이터가 적을수록 결과 편향 가능 → 최소 요건 강제 필요
- 교사·학부모에게 결과 과신 금지 안내 필수
- 학교생활기록부에 직접 인용 불가
