# 마음일기 업그레이드 계획 v4

> 작성일: 2026-07-10
> 목표: 학급 교우관계·갈등 파악을 위한 소시오메트리 조사 및 1페이지 리포트 도입

---

## 1. 개요

### 1.1 목적

기존 기능(감정기록, 계획, 평가, AI 성장 리포트)은 모두 **학생 개인 단위** 데이터를 다룬다. 이번 기능은 **학생 간 관계(누가 누구를 지명했는가)** 라는 새로운 데이터 축을 다뤄, 교사가 관찰만으로 파악하기 어려운 교우관계·고립 학생·갈등 조짐을 정기 설문으로 수집하고 한 장짜리 리포트로 시각화한다.

### 1.2 배경 — 왜 필요한가

- 교사가 30명 전후 학급의 관계망을 매일 관찰만으로 파악하기는 현실적으로 어렵다.
- 사회측정법(소시오메트리, Moreno)은 짧은 지명형 설문만으로 고립 학생·배타적 소그룹·인기 학생을 정량적으로 드러내는 검증된 방법이다.
- 감정 기록(개인 정서)과 결합하면 "이 학생이 최근 부정적 감정을 기록하면서 동시에 지명을 거의 못 받고 있다" 같은 조기 경보가 가능해진다 (v4 MVP 범위 밖, 향후 연계 여지로만 남겨둠).

### 1.3 민감도 등급

이 기능이 다루는 데이터는 앱 내에서 **가장 민감한 데이터**로 분류한다.

- 학생 상호 응답 원본(누가 누구를 지명했는지)은 **교사만** 조회 가능
- 학생 본인에게도 "내가 몇 표를 받았는지·누가 나를 어떻게 응답했는지"는 **절대 노출하지 않는다** (낙인·보복 위험)
- 부정 지명 문항은 기본값 OFF — 교사가 학급 상황에 맞게 opt-in

## 2. 설문 설계

### 2.1 문항 구성 (기본 템플릿)

| # | 유형 | 문항 예시 | 기본 노출 |
|---|------|-----------|-----------|
| 1 | 긍정 지명 (필수) | "함께 놀고 싶은/모둠하고 싶은 친구를 1~3명 골라주세요" | ON |
| 2 | 부정 지명 (선택) | "요즘 함께 하기 어려운 친구가 있다면 골라주세요" (저학년: "도움이 필요해 보이는 친구") | OFF (교사가 켜야 노출) |
| 3 | 역할 관찰형 (필수) | "우리 반에서 리더 역할을 하는 친구", "혼자 있는 것 같은 친구" 중 선택 응답 | ON |
| 4 | 개방형 (선택) | "요즘 반에서 기억에 남는 일이 있다면 적어주세요" (1~2문장) | ON |

### 2.2 학년군별 문구 조정

- 저학년(1~3학년): 부정 지명 문항은 기본 비활성 + 문구를 "도와주고 싶은/도움이 필요한"으로 순화
- 고학년(4~6학년): 부정 지명 활성 시에도 "왜 어려운지" 이유는 묻지 않음(익명 지명 수만 집계, 사유 텍스트는 남기지 않음)

### 2.3 MVP 범위

- v4는 **고정 템플릿**만 제공 (교사가 문항 자체를 새로 만드는 커스터마이즈는 범위 밖)
- 교사는 "부정 지명 포함 여부"만 토글로 선택

## 3. DB 스키마

```sql
-- 설문 회차 (학급당 여러 번 반복 실시 가능 — 학기 초/중/말 비교)
CREATE TABLE relationship_surveys (
  id                 UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id           UUID NOT NULL REFERENCES classes(id) ON DELETE CASCADE,
  teacher_id         UUID NOT NULL REFERENCES teacher_profiles(id) ON DELETE CASCADE,
  title              TEXT NOT NULL DEFAULT '교우관계 조사',
  includes_negative  BOOLEAN NOT NULL DEFAULT false,
  opened_at          TIMESTAMPTZ NOT NULL DEFAULT now(),
  closed_at          TIMESTAMPTZ,               -- NULL이면 응답 진행 중
  created_at         TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- 지명형 응답 (긍정/부정/역할) — 한 응답자가 문항당 여러 대상을 지명 가능
CREATE TABLE relationship_nominations (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  survey_id      UUID NOT NULL REFERENCES relationship_surveys(id) ON DELETE CASCADE,
  rater_id       UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,   -- 응답한 학생
  target_id      UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,   -- 지명된 학생
  question_type  TEXT NOT NULL CHECK (question_type IN ('positive', 'negative', 'role_leader', 'role_isolated')),
  created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (survey_id, rater_id, target_id, question_type)
);

-- 개방형 서술 응답
CREATE TABLE relationship_open_responses (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  survey_id   UUID NOT NULL REFERENCES relationship_surveys(id) ON DELETE CASCADE,
  rater_id    UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  content     TEXT NOT NULL,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (survey_id, rater_id)
);

-- 학생별 응답 완료 여부 (진행률 표시용)
CREATE TABLE relationship_survey_completions (
  survey_id   UUID NOT NULL REFERENCES relationship_surveys(id) ON DELETE CASCADE,
  student_id  UUID NOT NULL REFERENCES students(id) ON DELETE CASCADE,
  completed_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (survey_id, student_id)
);

ALTER TABLE relationship_surveys ENABLE ROW LEVEL SECURITY;
ALTER TABLE relationship_nominations ENABLE ROW LEVEL SECURITY;
ALTER TABLE relationship_open_responses ENABLE ROW LEVEL SECURITY;
ALTER TABLE relationship_survey_completions ENABLE ROW LEVEL SECURITY;
```

> RLS 정책은 기존 `students`/`eval_reports` 패턴과 동일하게 "교사는 본인 학급만", "학생은 본인 응답 제출만 가능(조회 불가)"으로 구성.

## 4. 집계 로직 (소시오그램 계산)

MVP는 그래프 이론을 깊게 쓰지 않고, 교사가 바로 이해할 수 있는 지표 위주로 단순하게 계산한다.

| 지표 | 계산 방법 | 의미 |
|------|-----------|------|
| 피지명 수 (인기도) | `question_type='positive'`인 row에서 `target_id` GROUP BY COUNT | 높을수록 또래 인기 높음 |
| 상호 지명 | A→B, B→A 긍정 지명이 모두 존재하는 쌍 | 실제 친밀한 관계로 해석 |
| 고립 학생 | 긍정 지명 피지명 수 = 0 | 우선 관찰 대상 |
| 갈등 위험 학생 | 부정 지명 피지명 수가 학급 평균 대비 현저히 높음 (예: 평균 + 2 이상) | 또래 갈등/따돌림 조짐 가능 |
| 소그룹 | 상호 지명 쌍을 엣지로 하는 단순 연결 요소(connected component) | 배타적 소그룹 여부 참고용 |

```
function buildSociogram(nominations):
  1. positive nominations로 방향 그래프 구성 (rater → target)
  2. 각 학생의 in-degree(피지명 수) 계산 → 인기도 랭킹
  3. A→B, B→A 동시 존재 쌍 추출 → 상호 지명 리스트
  4. 상호 지명 쌍을 엣지로 하는 connected component 계산 → 소그룹 리스트
  5. in-degree = 0인 학생 → 고립 학생 리스트
  6. negative nominations로 in-degree 계산 → 평균/표준편차 대비 이상치 → 갈등 위험 리스트
  7. 결과를 시각화용 JSON(nodes, edges, 지표)으로 반환
```

## 5. UI 설계

### 5.1 교사 대시보드 — "교우관계" 탭 신설

```
교사 대시보드
└── [학급관리] [평가] [편지함] [학급설정] [교우관계 ← 신규]
    │
    ├── 상단: 설문 회차 목록 (학기 초/중/말 비교 가능)
    │   └── [+ 새 설문 시작하기] 버튼
    │       └── 옵션: 부정 지명 포함 여부 토글
    │
    ├── 진행 중 설문: 응답 완료 X / 전체 Y명 진행률 표시
    │
    └── 완료된 설문 선택 → 리포트 화면
```

### 5.2 리포트 화면 (1페이지 요약)

```
┌───────────────────────────────────────────────┐
│ 3학년 2반 교우관계 리포트  (2026.07.10 실시)       │
├───────────────────────────────────────────────┤
│ [소시오그램]                                     │
│   ○ 학생 노드 (피지명 수에 비례해 크기 조정)         │
│   → 화살표: 지명 방향 (상호 지명은 굵은 실선)        │
│                                                 │
│ [핵심 지표]                                      │
│  고립 학생: 2명   상호 지명 쌍: 9쌍   소그룹: 3개     │
│                                                 │
│ [주의 학생 목록]                                  │
│  · 7번 김OO — 긍정 지명 0회 (고립)                 │
│  · 15번 이OO — 부정 지명 4회 (평균 대비 높음)        │
│                                                 │
│ [개방형 응답 요약]                                 │
│  (선택) AI가 서술형 응답을 종합해 주의 깊게 볼 지점 요약 │
│                                                 │
│ ⚠ 이 리포트는 참고 자료이며, 학생에게 결과를          │
│   공개하거나 생활기록부에 직접 인용하지 않습니다.       │
├───────────────────────────────────────────────┤
│                                    [PDF 저장]    │
└───────────────────────────────────────────────┘
```

- 소시오그램은 단순 원형 배치(학생 수만큼 원 둘레에 배치) + SVG 화살표로 MVP 구현 — force-directed 레이아웃 라이브러리 도입은 범위 밖(추후 검토)
- PDF 저장은 기존 리포트 PDF 인프라(성장 리포트, AboutMe 보고서와 동일한 방식) 재사용

### 5.3 학생 응답 화면

- 학생 로그인 후 대시보드 상단에 배너 형태로 "교우관계 설문에 참여해주세요" 노출 (진행 중 설문이 있고 미응답 상태일 때만)
- 문항별로 학급 명단에서 체크박스로 선택 (본인 제외)
- 부정 지명 문항은 `includes_negative=true`일 때만 노출
- 제출 후에는 재응답 불가(수정하려면 교사에게 요청) — 결과 확인 화면 없음(자신의 응답 요약도 보여주지 않음, 실수로도 다른 학생 결과 유추 방지)

## 6. API 설계

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/relationship/surveys` | 교사 — 새 설문 생성 (class_id, includes_negative) |
| GET | `/api/relationship/surveys?classId=` | 교사 — 설문 회차 목록 + 진행률 |
| GET | `/api/relationship/surveys/[id]` | 학생 — 응답 폼에 필요한 문항/학급 명단 조회 (본인 제외) |
| POST | `/api/relationship/surveys/[id]/responses` | 학생 — 지명 + 개방형 응답 제출 (1회 한정) |
| POST | `/api/relationship/surveys/[id]/close` | 교사 — 설문 마감 |
| GET | `/api/relationship/surveys/[id]/report` | 교사 — 집계된 소시오그램 + 지표 + 주의 학생 목록 조회 |

`report` 엔드포인트는 `requireTeacher()` + 학급 소유 검증 후 `buildSociogram()` 결과를 반환하며, 학생 응답 원본(누가 누구를 지명했는지 raw row)은 별도 API로 노출하지 않는다 — 리포트는 항상 집계된 지표 형태로만 내려간다.

## 7. (선택) AI 서술 요약

개방형 응답과 지표를 종합해 "이 학급에서 주의 깊게 볼 지점"을 gpt-4o로 한 문단 요약한다. v3에서 구축한 인프라(`lib/ai/*`, 익명화, `store: false`)를 그대로 재사용한다.

- 프롬프트에는 실명 대신 출석번호 코드만 포함 (v3 2.1 원칙 동일 적용)
- 학생 원문 서술은 그대로 전송하되, 특정 학생을 저격하는 지도 지시("OO를 혼내세요" 등)는 생성하지 않도록 시스템 프롬프트에 명시
- 생성 결과는 교사에게만 노출, PDF 리포트에도 "AI 참고 요약" 라벨과 함께 포함 가능
- **MVP에서는 생략 가능한 부가 기능** — 소시오그램 지표만으로도 리포트 가치는 충분하므로 Phase 4(선택)로 분리

## 8. 개인정보·윤리 체크리스트

- [ ] 학생 상호 응답 raw 데이터는 교사 전용 API에서만 조회 가능한지 (학생 세션으로 접근 불가 확인)
- [ ] 학생 본인에게도 득표 결과·지명 상세를 보여주지 않는지 (실수로 노출되는 화면 없는지 전수 확인)
- [ ] 부정 지명 문항 기본값이 OFF인지, 저학년 문구가 순화되어 있는지
- [ ] 부정 지명에 "사유" 자유서술을 요구하지 않는지 (사유 텍스트는 낙인·유출 위험이 커서 수집 자체를 금지)
- [ ] 리포트/PDF에 "참고 자료이며 생활기록부 직접 인용 금지" 문구가 항상 표시되는지
- [ ] AI 요약 사용 시 학생 실명 미전송, `store: false` 적용 여부 (v3 체크리스트와 동일 기준)
- [ ] 설문 응답은 1회 제출 후 잠금 처리되어 있는지 (재제출로 인한 데이터 오염 방지)

## 11. 한계 및 주의사항

- 저학년은 문해력·판단 기준이 불안정해 지명 결과의 신뢰도가 낮을 수 있음 — 학년별로 결과 해석에 참고 수준 차등 부여 권장
- 소시오그램은 "한 시점의 스냅샷"일 뿐이며, 학기 초/중/말 반복 실시로 변화 추이를 함께 봐야 신뢰도가 높아짐
- 결과를 단정적으로 해석해 특정 학생에게 낙인을 찍는 용도로 쓰지 않도록 교사 안내 문구 필수
- 부정 지명 데이터는 학교폭력 등 공식 절차의 증거 자료로 사용할 수 없음(참고 자료임을 명확히 고지)
