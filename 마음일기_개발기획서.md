# 🌱 마음일기 (MaumDiary) — 개발 기획서

> **초등학생 감정 피드 & 일일 계획 관리 SaaS**
> 작성일: 2026.02.27 | 최종 수정: 2026.04.21 | 버전: v1.4

---

## 1. 프로젝트 개요

### 1.1 서비스 정의

"마음일기(MaumDiary)"는 초등학교 교사가 학급을 운영하며, 학생들의 **감정 표현**과 **일일 계획 실천**을 체계적으로 관리할 수 있는 교육용 SaaS 플랫폼이다.

### 1.2 핵심 가치

- **감정 공유**: 학생들이 매일 자신의 감정을 인스타그램 스타일의 피드로 공유하고, 같은 학급 친구들과 소통한다.
- **계획 실천**: 학생 스스로 세운 계획을 매일 체크하며 자기주도 습관을 형성한다.
- **교사 모니터링**: 교사가 학급 전체와 개별 학생의 감정 상태 및 계획 달성률을 한눈에 파악한다.
- **관계 회복**: 학생들이 같은 학급 친구에게 편지를 보내며 따뜻한 관계를 쌓고, 교사가 학급 편지를 관리한다.

### 1.3 대상 사용자

| 사용자 | 설명 |
|--------|------|
| **교사** | 학급 개설, 학생 계정 관리, 감정 피드 확인, 계획 달성률 모니터링, 학급 편지 열람·수정·삭제 |
| **학생** | 감정 피드 작성/열람, 일일 계획 체크, 본인 달성률 확인, 학급 친구에게 편지 보내기·받기 |

## 2. 사용자 흐름 (User Flow)

### 2.1 교사 흐름

```
회원가입/로그인
    └─→ 학급 생성 (학급명, 학년/반)
         └─→ 학생 계정 생성 (이름, 번호 입력)
              └─→ 대시보드
                   ├─→ [감정 피드 탭] 학생 감정 피드 전체 열람 & 코멘트
                   ├─→ [계획 관리 탭] 학생별 계획 달성 현황 확인
                   ├─→ [관계회복의 날 탭] 학급 내 편지 전체 열람 & 수정/삭제
                   └─→ [통계 탭] 개별/전체 달성률 그래프 확인
```

### 2.2 학생 흐름

```
로그인 (학급코드 + 이름)
    └─→ 홈 화면
         ├─→ [오늘의 감정] 감정 피드 작성 후 저장 → 달력에서 날짜 선택 → 해당 날짜의 내 감정(선택 감정 + 작성 내용) 조회
         ├─→ [감정 타임라인] 같은 반 친구들 피드 열람 & 응원 반응
         ├─→ [오늘의 계획] 날짜 선택 후 해당 날짜 계획 조회 → 오늘만 체크/추가/삭제 가능
         ├─→ [편지함] 받은 편지 목록 열람 & 친구에게 편지 쓰기
         └─→ [나의 기록] 월간 달성률 그래프 확인
```

## 3. 주요 기능 상세

### 3.1 교사 기능

#### 3.1.1 회원가입 / 로그인

- 이메일 + 비밀번호 기반 회원가입
- 소셜 로그인 (Google) 지원
- 비밀번호 재설정 기능

#### 3.1.2 학급 관리

- 학급 생성: 학급명, 학년, 반, 학급코드 입력 (교사당 1개만 생성 가능)
- 학급코드: 숫자 1~6자리 형식, 전체 시스템에서 중복 불가 (학생 로그인 시 사용)
- 학급 정보 수정 / 학급 삭제 (기간 제한 없이 즉시 가능)
- 단일 학급 운영 (학급 삭제 후 재생성 가능)

#### 3.1.3 학생 계정 관리

- 학생 등록: **이름 + 출석번호** 입력
- 일괄 등록: CSV 업로드 지원
- 학생 정보 수정 / 삭제

#### 3.1.4 감정 피드 모니터링

- 학급 전체 감정 피드 타임라인 열람
- 날짜별 타임라인 조회 (기본: 오늘)
- 우측 상단 달력에서 날짜 선택 후 해당 날짜 피드 조회
- 학생별 감정 피드 필터링
- 감정 범주별 필터 (기쁨/활력, 애정/유대, 불안/긴장, 슬픔/무기력, 분노/거부, 사회적 감정)
- 교사 전용 댓글 작성 기능 (학생 작성 불가)
- 교사 댓글은 해당 피드 작성 학생 본인에게만 노출
- 부적절한 피드 비공개 처리 기능
- **감정 알림**: 부정 정서(불안/긴장, 슬픔/무기력, 분노/거부)가 연속 3일 이상 지속되면 교사에게 알림

#### 3.1.5 계획 달성 모니터링

- 학생별 계획 등록 현황 확인
- 학생별 일일 체크 현황 확인
- 전체 학급 오늘의 달성률 요약
- **월간 리포트**: 학생별 / 전체 달성률 그래프

#### 3.1.6 평가 피드백 관리

##### 채점기준 관리

- 교사는 학급 단위로 **채점기준(루브릭)** 을 미리 목록화하여 등록
- 채점기준 1개는 **기준명(제목) + 공통 설명 + 복수의 평가기준**으로 구성

  | 항목 | 설명 | 최대 길이 |
  |------|------|----------|
  | **기준명** | 채점기준 제목 (예: "발표 태도") | 100자 |
  | **도달목표** | 이 기준으로 학생이 달성해야 할 목표 | 200자 |
  | **수행과제** | 평가에 사용되는 구체적인 과제 또는 활동 | 200자 |
  | **평가기준** (복수) | 세부 평가항목 목록 (예: "내용 전달력", "목소리 크기") — 각 기준마다 잘함/보통/노력 수준 기술 | 기준명 100자, 수준 각 200자 |

- 평가기준은 1개의 채점기준 안에 여러 개 추가 가능 (JSONB 배열로 저장)
- 각 평가기준에 **잘함 / 보통 / 노력** 수준 기술 입력 가능 (선택)
- 기준 순서 변경 / 수정 / 삭제 가능
- 등록된 기준은 평가 작성 시 재사용 (별도 재입력 불필요)
- **채점기준 고정(Pin)** 기능: 자주 쓰는 기준을 고정해 두면 학생 클릭 시 바로 평가 폼 오픈

##### 학생별 평가 작성

- 교사 대시보드 **평가피드백 탭** 에서 학생을 선택한 뒤 평가보고서 작성
- 채점기준 선택 → 각 평가기준마다:
  - 수준 기술을 **읽기 전용**으로 표시 (참고용)
  - **잘함 / 보통 / 노력** 3단계 등급 버튼 선택 (기존 상/중/하에서 변경)
  - 해당 학생에 대한 짧은 **개별 피드백** 텍스트 입력 (선택, 최대 200자)
- **평가 자료 이미지 업로드** (선택, 보고서당 최대 5장) — 평가 작성 시 함께 첨부
  - 학생의 수행 결과물 사진, 작성 자료 등을 첨부
  - 지원 형식: JPG, PNG, WEBP / 장당 최대 10MB
  - Supabase Storage에 저장, 서명된 URL(signed URL)로만 접근 허용
  - **다운로드 불가**: `Content-Disposition: inline` 헤더 설정, 우클릭/드래그 방지 처리
- **웹 링크 첨부** (선택, 보고서당 최대 10개) — 평가 작성 시 함께 입력
  - 참고 자료 URL 첨부 (링크명 + 주소), `https://` 미입력 시 자동 추가
  - 학생 화면에서 클릭 시 새 탭으로 열기
- **일괄 저장**: 등급/피드백/이미지/링크를 모두 입력 후 **"평가 저장"** 버튼 1회 클릭으로 저장
- **저장 후 수정**: 보고서 상세 모달에서 **수정** 버튼으로 등급/피드백 변경 가능 (이미지/링크는 상세 모달에서 추가·삭제)

##### 접근 권한

- 교사만 평가보고서 작성·수정·삭제 가능
- 학생은 자신의 보고서만 조회 가능 (타 학생 열람 불가)
- 부모님 응원은 학생 화면에서만 입력 가능 (별도 계정 없음)

#### 3.1.7 관계회복의 날 (학급 편지 관리)

- 교사 대시보드에 **관계회복의 날** 탭을 별도 제공
- 학급 내 학생들이 주고받은 편지 **전체 목록** 열람
  - 목록 표시 항목: 발신자(보낸 학생), 수신자(받는 학생), 제목, 작성일, 읽음 여부
  - 최신순 정렬
  - 제목 클릭 시 **편지 상세 모달**로 내용 확인
- **편지 수정**: 부적절한 내용을 교사가 직접 수정 가능 (제목 + 내용 모두 수정)
- **편지 삭제**: 부적절한 편지를 교사가 삭제 가능 (삭제 시 발신자·수신자 편지함에서도 제거)
- 삭제된 편지는 복구 불가하므로 삭제 전 확인 다이얼로그 표시

#### 3.1.8 통계 대시보드

- 교사 대시보드 탭 구성: **학급 관리 / 학생 관리 / 마음피드 / 평가피드백 / 통계 대시보드**
- **평가피드백 탭 구성**
  - 왼쪽 패널: 학생 목록 (이름/번호) — 클릭 시 해당 학생 평가 목록 표시
  - 오른쪽 패널: 선택 학생의 평가보고서 목록 + 새 평가 작성 버튼
  - 보고서 목록: 제목 / 작성일 / 항목 수 표시, 클릭 시 상세 모달
  - 보고서 상세 모달: 채점기준별 등급(상/중/하) + 피드백, 학생 성찰일기(읽기 전용), 부모님 응원(읽기 전용)
- **채점기준 관리** 서브 탭: 기준 추가/수정/삭제/순서 변경
- **학생 관리 탭 카드형 UI**
  - 학생 추가 시 카드가 즉시 목록에 반영
  - 데스크톱 기준 한 줄에 3개 카드(모바일은 반응형 축소)
  - 카드 내 번호/이름/오늘 계획 달성률 표시
  - 학생이 오늘 일일계획을 모두 완료 선택한 경우 카드 배경을 초록색으로 강조
- **마음피드 탭**에서 학생 감정피드 모니터링
  - 날짜 선택(기본: 오늘) 후 해당 날짜 피드만 조회
  - 최신순 카드 목록 (학생 번호/이름, 감정 범주/세부 감정, 내용, 작성 시각)
  - 피드별 반응 수(하트/응원/힘내) 확인
- **학생 카드 뷰**: 등록된 학생을 카드형 목록으로 표시 (번호/이름)
- 카드 클릭 시 **상세 통계 창(모달)** 오픈
  - 오늘 실천률
  - 계획별 실천률
  - 감정 분포도
- 기간 필터: 주간 / 월간 / 학기 단위
- 상세 통계 창 기준 PDF 내보내기 기능

### 3.2 학생 기능

#### 3.2.1 로그인

- **학급코드 + 이름** 으로 간편 로그인
- 학급코드는 숫자 1~6자리만 허용
- 비밀번호/PIN 없이 접속
- 자동 로그인 유지 (기기별)

#### 3.2.2 감정 피드 (인스타그램 스타일)

- **피드 작성**
  - 감정 범주 선택 후 세부 감정 선택 (부록 A 범주형 분류 사용)
  - 한 줄 텍스트 입력 (최대 100자)
  - 이미지 첨부 (선택, 최대 1장)
  - 하루 최대 1개 피드 작성 가능
- **내 감정 저장 / 날짜별 조회**
  - 학생이 선택한 감정(범주/세부 감정)과 작성한 한 줄 기록을 해당 날짜 기준으로 저장
  - `오늘의 감정` 화면 상단 달력에서 날짜 선택 가능
  - 선택한 날짜에 저장된 내 감정 기록이 있으면 감정 범주, 세부 감정, 작성 내용, 작성 시각을 조회
  - 과거 날짜의 감정 기록은 읽기 전용이며 수정 불가
  - 오늘 날짜에 아직 기록이 없을 때만 새 감정 작성 가능
- **피드 타임라인**
  - 같은 학급 친구들의 감정 피드 열람 (인스타그램형 카드 UI)
  - 기본은 오늘 날짜 피드만 노출
  - 우측 상단 달력에서 날짜 선택 시 해당 날짜 피드만 조회
  - 최신순 정렬
  - 응원 반응 보내기 (하트, 응원, 힘내 — 3종)
  - 교사 댓글은 내 피드에 달린 내용만 확인 가능
- **내 피드 관리**
  - 본인 피드 삭제 가능
  - 본인 감정 기록 히스토리 확인

#### 3.2.3 일일 계획 관리

- **계획 등록**
  - 학생 본인이 직접 계획 등록 (예: "책 30분 읽기", "줄넘기 100개")
  - 최대 5개 계획 등록 가능
  - 계획 제목 수정 / 삭제 (당일에만 가능)
  - **계획 제목 수정 시 기존 체크 기록(plan_checks)은 plan_id로 연동되어 유지됨**
  - **변경 이력 토글**: 각 계획 우측의 "변경 이력" 버튼 클릭 시 수정 전/후 제목과 날짜를 조회 가능
- **매일 리프레시**
  - 매일 자정(00:00) 기준 체크 상태 초기화
  - 등록된 계획 목록은 유지, 체크만 리셋
  - 오늘 날짜의 계획 체크 화면이 기본 노출
- **날짜별 조회 / 수정 제한**
  - 학생 대시보드의 계획 탭에서 달력으로 날짜 선택 가능
  - 선택한 날짜에 저장된 완료/미완료/미선택 상태를 그대로 조회
  - 과거 날짜의 데이터는 읽기 전용이며 수정 불가
  - 계획 추가/삭제/체크 변경은 오늘 날짜에서만 허용
- **계획 체크**
  - 각 계획 항목에 **완료 / 미완료** 버튼 제공
  - 초기 상태는 **미선택(null)** 으로 표시되며 학생이 직접 선택
  - 완료 선택: 완료 처리, 미완료 선택: 미완료 처리, 다시 누르면 미선택으로 해제 가능
  - 완료 시 작은 축하 애니메이션
  - 오늘의 달성률 표시 (예: 3/5 완료 선택 — 60%)

#### 3.2.4 나의 기록 (달성률 그래프)

- 월간 달성률 캘린더 히트맵 (달성률에 따라 색상 변화)
- 계획별 월간 달성률 막대 그래프
- 주간 추이 라인 차트
- 연속 달성일 카운트 표시 (스트릭)

#### 3.2.5 평가기록 (책장 뷰)

- **책장(Bookshelf) 스타일 목록**: 교사에게 받은 평가보고서를 책등처럼 카드로 나열
  - 카드에 보고서 제목 + 작성일 표시
  - 카드 클릭 시 **보고서 상세 모달** 오픈
- **보고서 상세 모달 구성**:
  1. **교사 평가 영역** (읽기 전용)
     - 채점기준별 등급(잘함/보통/노력) 배지 + 교사 피드백 텍스트
     - 채점기준의 도달목표 / 수행과제 / 해당 등급 수준 기술 확인 가능 (토글 열기)
     - **← 내 수준** 하이라이트: 내가 받은 등급의 수준 기술을 강조 표시
  2. **평가 자료 이미지** (읽기 전용)
     - 교사가 첨부한 이미지를 썸네일 갤러리로 표시
     - 썸네일 클릭 시 **라이트박스(전체화면 모달)**로 크게 보기
     - 우클릭 / 드래그 비활성화, 다운로드 버튼 없음
     - 이미지는 signed URL로만 제공 (직접 URL 접근 차단)
  3. **성찰일기 작성칸** (학생 전용)
     - 교사 평가 아래 별도 구역
     - 최초 1회 작성 가능 (작성 후 수정 불가, 신중하게 쓰도록 유도)
     - 최대 500자
  4. **부모님 응원 / 격려 칸**
     - 성찰일기 아래 별도 구역
     - 최초 1회 입력 가능 (수정 불가)
     - 최대 300자
     - 별도 계정 없이 학생 화면에서 직접 입력

#### 3.2.6 학급 편지함

- **편지 쓰기**
  - 수신자: 같은 학급코드 안의 학생 중 1명 선택 (본인 제외)
  - 제목 입력 (최대 50자)
  - 내용 입력 (최대 1000자)
  - 전송 버튼 클릭 시 수신자의 편지함으로 전달
- **받은 편지함**
  - 지메일 스타일 목록: 발신자 이름 + 제목 + 작성일 표시 (내용은 목록에 미표시)
  - 최신순 정렬
  - 읽지 않은 편지는 굵은 글씨 + 새 편지 뱃지로 강조
  - 제목 클릭 시 **편지 상세 모달**로 전체 내용 확인 (읽음 처리)
- **보낸 편지함**
  - 내가 보낸 편지 목록: 수신자 이름 + 제목 + 작성일 표시
  - 제목 클릭 시 상세 모달로 내용 확인
- **편지 상세 모달**
  - 발신자, 수신자, 제목, 내용, 작성일 표시
  - 작은 창(모달) 형태로 오픈
  - 받은 편지 기준 읽음 상태 자동 업데이트

### 3.3 학부모 참여 기능

- 별도 계정/로그인 없이 학생 대시보드 내 **부모님 응원칸**을 통해 참여
- 응원 메시지는 해당 평가보고서에 1회 연결되며, 교사도 보고서 상세에서 확인 가능
- 향후 학부모 전용 읽기 전용 페이지 확장 가능 (Phase 4 이후 검토)

```
  - created_at

[Class]
  - id (PK)

  - created_at

[FeedReaction]
  - id (PK)
  - feed_id (FK → EmotionFeed)

  - UNIQUE(feed_id, student_id)  -- 피드당 1인 1반응

[Plan]

[PlanTitleHistory]

  - plan_id (FK → Plan)

  - check_date (DATE)
  - is_completed (BOOLEAN, NULL 허용: TRUE=완료 / FALSE=미완료 / NULL=미선택)
  - checked_at (TIMESTAMP)
  - UNIQUE(plan_id, check_date)  -- 날짜별 1회 체크

[TeacherComment]
  - id (PK)
  - teacher_id (FK → Teacher)
  - feed_id (FK → EmotionFeed)
  - content (VARCHAR 200)
  - created_at

[EvalRubric]  -- 채점기준 (교사가 미리 등록)
  - id (PK)
  - teacher_id (FK → Teacher)
  - title (VARCHAR 100)              -- 기준명 (예: "발표 태도")
  - goal (VARCHAR 200, NULLABLE)     -- 도달목표
  - task (VARCHAR 200, NULLABLE)     -- 수행과제
  - criteria (JSONB, DEFAULT '[]')   -- 복수 평가기준 배열
    -- 항목 구조: { title: string, level_high: string|null, level_mid: string|null, level_low: string|null }
  - sort_order (INT)
  - is_active (BOOLEAN)
  - created_at

[EvalReport]  -- 평가보고서 (학생 1명에 대한 1건의 평가)
  - id (PK)
  - student_id (FK → Student)
  - teacher_id (FK → Teacher)
  - title (VARCHAR 100)         -- 보고서 제목 (예: "1학기 중간 평가")
  - created_at

  - id (PK)
  - report_id (FK → EvalReport)
  - rubric_id (FK → EvalRubric, NULLABLE)         -- 기준 삭제 시 null 허용
  - rubric_title_snapshot (VARCHAR 100)            -- 작성 시점 채점기준 제목 스냅샷
  - rubric_goal_snapshot (VARCHAR 200, NULLABLE)
  - rubric_task_snapshot (VARCHAR 200, NULLABLE)
  - rubric_level_high_snapshot (VARCHAR 200, NULLABLE)  -- 해당 평가기준의 잘함 수준 기술
  - rubric_level_mid_snapshot (VARCHAR 200, NULLABLE)   -- 보통
  - rubric_level_low_snapshot (VARCHAR 200, NULLABLE)   -- 노력

  - grade (ENUM: high/mid/low)                     -- 잘함/보통/노력
  - teacher_feedback (VARCHAR 200, NULLABLE)       -- 교사 개별 피드백
  - sort_order (INT)
  - created_at

[EvalReportLink]  -- 평가 참고 웹 링크 (보고서당 최대 10개)
  - id (PK)
  - report_id (FK → EvalReport)
  - url (TEXT)                  -- 참고 자료 URL

[EvalReportImage]  -- 평가 자료 이미지 (보고서당 최대 5장)

  - sort_order (INT)
  - created_at

[EvalReflection]  -- 학생 성찰일기
  - id (PK)

[EvalParentComment]  -- 부모님 응원/격려

[Letter]  -- 학급 편지 (학생 간 편지)

  - sender_id (FK → Student)            -- 보낸 학생
  - recipient_id (FK → Student)         -- 받는 학생
  - title (VARCHAR 50)                  -- 편지 제목
  - content (TEXT, MAX 1000자)          -- 편지 내용
  - is_read (BOOLEAN, DEFAULT false)    -- 수신자 읽음 여부
  - created_at
  - updated_at                          -- 교사 수정 시 갱신
```

### 5.2 주요 관계

- Teacher 1 : 0..1 Class (교사 한 명이 최대 1개 학급 운영)
- Class 1 : N Student (학급 하나에 여러 학생)
- Student 1 : N EmotionFeed (학생 한 명이 여러 감정 피드)
- Student 1 : N Plan (학생 한 명이 여러 계획)
- Plan 1 : N PlanCheck (계획 하나에 날짜별 체크 기록)
- Plan 1 : N PlanTitleHistory (계획 하나에 여러 제목 변경 이력)
- EmotionFeed 1 : N FeedReaction (피드 하나에 여러 반응)
- EmotionFeed 1 : N TeacherComment (피드 하나에 교사 코멘트)
- Teacher 1 : N EvalRubric (교사가 여러 채점기준 등록)
- Student 1 : N EvalReport (학생 한 명에 여러 평가보고서)
- EvalReport 1 : N EvalReportItem (보고서 하나에 기준별 항목)
- EvalReport 1 : N EvalReportImage (보고서 하나에 최대 5장 이미지)
- EvalReport 1 : N EvalReportLink (보고서 하나에 최대 10개 웹 링크)
- EvalReport 1 : 0..1 EvalReflection (보고서 하나에 학생 성찰일기 1건)
- EvalReport 1 : 0..1 EvalParentComment (보고서 하나에 부모님 응원 1건)
- Student 1 : N Letter (as sender, 보낸 편지)
- Student 1 : N Letter (as recipient, 받은 편지)
- Class 1 : N Letter (학급 내 편지 전체)

## 8. 보안 및 개인정보 보호

### 8.1 아동 개인정보 보호

- **최소 정보 수집 원칙**: 이름, 출석번호만 수집 (이메일, 전화번호 수집 안 함)
- **보호자 동의**: 서비스 도입 시 학급 단위 보호자 동의서 절차 안내
- **데이터 보존 기간**: 학기 종료 후 3개월 뒤 자동 삭제 또는 교사 수동 삭제
- **아동 개인정보보호법(개인정보보호법 제39조의3)** 준수

### 8.2 접근 제어

- 교사: 본인 학급의 데이터에만 접근 가능 (Supabase RLS)
- 학생: 본인 학급의 감정 피드만 열람 가능, 타 학급 접근 차단
- 계획 데이터: 본인 것만 열람/수정 가능 (교사는 조회만 가능)

### 8.3 콘텐츠 안전

- 피드 텍스트 금칙어 필터링 (욕설, 부적절한 표현)
- 이미지 업로드 시 파일 형식/크기 제한 (JPG, PNG, WEBP / 감정피드 최대 5MB, 평가자료 장당 최대 10MB)
- 교사 피드 비공개 처리 권한

### 8.4 평가 자료 이미지 보호

- 평가 자료 이미지는 **Supabase Storage** private 버킷에 저장 (공개 URL 없음)
- 접근 시 서버에서 **signed URL** 발급 (유효 시간: 10분)
- 응답 헤더 `Content-Disposition: inline` 설정 → 브라우저에서 열기만 가능, 저장 불가
- 학생 화면에서 이미지에 `pointer-events: none` + `onContextMenu` 차단으로 우클릭/드래그 방지
- 다운로드 버튼/링크 미제공
- 교사 본인 학급 학생의 보고서 이미지만 접근 가능 (타 학급 차단)

## 11. 핵심 비즈니스 규칙 정리

| # | 규칙 |
|---|------|
| 1 | 학생 계정은 교사만 생성할 수 있다 |
| 2 | 학생은 하루 최대 1개의 감정 피드를 작성할 수 있다 |
| 3 | 학생은 최대 5개의 계획을 등록할 수 있다 |
| 4 | 계획의 체크 상태(완료/미완료/미선택)는 매일 자정(00:00) 초기화된다 |
| 5 | 계획 목록 자체는 학생이 삭제하지 않는 한 유지된다 |
| 6 | 감정 피드는 교사만 전체 열람 가능하며, 학생은 본인 피드만 조회할 수 있다 |
| 7 | 교사만 피드에 댓글을 작성할 수 있으며, 해당 댓글은 피드 작성 학생 본인에게만 노출된다 |
| 8 | 교사는 부적절한 피드를 비공개 처리할 수 있다 |
| 9 | 부정적 감정 3일 연속 시 교사에게 알림이 발송된다 |
| 10 | 1개월 이상 데이터 축적 시 달성률 그래프가 활성화된다 |
| 11 | 교사는 학급을 기간 제한 없이 즉시 삭제할 수 있다 |
| 12 | 피드 타임라인은 선택한 날짜의 피드만 조회한다 (기본값: 오늘) |
| 13 | 교사는 학급을 1개만 생성할 수 있으며, 기존 학급이 있으면 생성 UI가 비활성화된다 |
| 14 | 학급코드는 숫자 1~6자리여야 하며, 동일한 코드로는 생성할 수 없다 |
| 15 | 채점기준은 교사가 미리 등록하며, 1개 채점기준 안에 복수의 세부 평가기준(criteria)을 JSONB 배열로 구성할 수 있다 |
| 16 | 평가 등급은 잘함/보통/노력(내부값 high/mid/low)으로 표시하며, 교사만 작성·수정·삭제 가능하다. 학생은 본인 보고서만 읽기 전용으로 조회할 수 있다 |
| 17 | 성찰일기는 학생이 해당 평가보고서에 최초 1회만 작성 가능하며, 작성 후 수정할 수 없다 |
| 18 | 부모님 응원은 해당 평가보고서에 최초 1회만 입력 가능하며, 별도 계정 없이 학생 화면에서 작성한다 |
| 19 | 계획 제목 수정 시 기존 체크 기록(plan_checks)은 plan_id로 연동되어 그대로 유지된다 |
| 20 | 계획 제목 변경 이력은 변경 시 자동으로 plan_title_history에 기록되며, 학생이 조회할 수 있다 |
| 21 | 평가보고서 작성 시 이미지 첨부(최대 5장)·웹 링크(최대 10개)·등급·피드백을 모두 입력 후 한 번에 저장한다 |
| 22 | 웹 링크 입력 시 `https://` 프로토콜이 없으면 자동으로 추가된다 |
| 23 | 교사는 저장된 보고서의 등급과 피드백을 상세 모달에서 수정할 수 있다 (이미지·링크는 상세 모달에서 추가·삭제) |
| 24 | 편지는 같은 학급코드 안의 학생에게만 보낼 수 있으며, 본인에게는 보낼 수 없다 |
| 25 | 편지 제목은 최대 50자, 내용은 최대 1000자이다 |
| 26 | 받은 편지는 상세 모달을 열면 읽음 상태로 자동 처리된다 |
| 27 | 교사는 학급 내 모든 편지를 열람·수정·삭제할 수 있으며, 학생은 본인이 받은 편지와 보낸 편지만 조회할 수 있다 |
| 28 | 교사가 편지를 삭제하면 발신자·수신자 편지함 모두에서 즉시 제거되며 복구할 수 없다 |

| **히트맵** | 캘린더 위에 달성률을 색상 농도로 표현한 시각화 (GitHub 잔디 스타일) |
| **편지함** | 학급 내 학생 간 주고받는 편지를 보관하는 기능; 받은편지함·보낸편지함으로 구분 |
| **관계회복의 날** | 교사 대시보드에서 학급 전체 편지를 열람·수정·삭제하는 관리 탭 |
